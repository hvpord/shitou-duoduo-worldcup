## 数据位置
在 `groups.html` 的 `<script>` 标签中，找到 `groupsData` 数组。结构如下：

```javascript
const groupsData = [
    {
        name: 'A 组',
        teams: [
            { name: '墨西哥', flag: '🇲🇽', points: 3, played: 1 },
            { name: '韩国', flag: '🇰🇷', points: 3, played: 1 },
            ...
        ]
    },
    ...
];
```

每个球队对象包含：
- `name`：球队中文名（必须与页面上显示的一致，否则选队功能会失效）
- `flag`：国旗 emoji
- `points`：当前积分
- `played`：已赛场次

## 更新步骤

1. **查询最新赛果和积分**
   - 优先使用 ESPN 官方积分榜：`https://www.espn.com/soccer/standings/_/league/fifa.world`
   - 也可参考 FIFA 官网或其他权威体育媒体
   - 注意：2026 年世界杯为 48 队 12 组（A–L），每组 4 队

2. **获取或确认 `groups.html` 文件**
   - 如果本地有 `/Users/zhanghui/zhanghuihub/家庭/育儿/groups.html`，用它作为编辑基础
   - 如果本地没有，从 GitHub 仓库克隆到临时目录

3. **修改 `groupsData` 数组**
   - 逐组更新每支球队的 `points` 和 `played`
   - 不要修改 `name` 字段，否则之前石头和多多选队的记录会丢失
   - 不要修改 `flag` 字段

4. **更新页面上的日期**
   - 找到页面中的「积分更新于：2026 年 X 月 X 日」文字
   - 共有两处：信息框（`.info-box`）和底部说明（`.status-note`）
   - 改成当前日期

5. **本地验证**
   - 用浏览器打开编辑后的 `groups.html`，确认 12 个小组、48 支球队、积分都正确显示
   - 确认 HTML 没有语法错误，脚本标签正确闭合

6. **同步到 GitHub**
   - 如果使用本地文件：将修改后的 `groups.html` 复制到 GitHub 仓库（本地 clone 或直接推送到 GitHub）
   - 如果使用临时 clone：直接在该目录提交并推送
   - 推送到 `main` 分支
   - 等待 1–2 分钟后，访问 `https://hvpord.github.io/shitou-duoduo-worldcup/groups.html` 确认已生效

## 注意事项
- 该页面使用浏览器 `localStorage` 和 URL hash 保存石头、多多的选队记录
- 球队中文名是选队功能的关键标识，**绝对不能改**
- 如果只更新比分、不改动队名，孩子们之前的选择会自动保留
- 页面有缓存，更新后如果看不到最新内容，按 `Ctrl + Shift + R` 强制刷新
- 如果本地 Obsidian 仓库也有这个文件，更新 GitHub 后建议把最终版本也同步回 `/Users/zhanghui/zhanghuihub/家庭/育儿/groups.html`，保持本地和线上一致
